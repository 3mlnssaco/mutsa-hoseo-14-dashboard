🌟 핵심 변화: PBL 10주차
이제 PBL 10주차는 미니프로젝트가 아닌 RISE 프로젝트 설계로 변경되었습니다. 6월 2주에 집중적으로 MVP 정의, ERD, API 명세, 기술 스택, 백로그를 완성하고, 6월 말 킥오프부터 7-8월 스프린트로 들어갑니다.

🤝 팀매칭 & RISE 프로젝트 파이프라인

📌 8단계: 팀빌딩 → 아이디어톤 → 사업계획서 → RISE → 프로젝트 설계
팀을 먼저 만들고, 팀 단위로 아이디어톤에 나가고, 사업계획서를 써서 RISE에 신청합니다.
최종 설계 단계(6월 2주)에서 MVP, 기술스택, 백로그를 완성하여 본격 개발을 시작합니다.
①
팀빌딩 (5주, 3/30~)
욕심 레벨 설문 → A/B/C 매칭 → 팀 확정 + 역할 분담 (PM/FE/BE/Design)
②
팀별 아이디어 발굴 (6주, 4/6~)
팀 단위 브레인스토밍 + 문제 정의 + 아이디어 후보 리스트 작성
③
교내 아이디어톤 (7주, 4/13~)
교내 팀 단위 피치 → 투표 + 피드백. 아이디어 1pager 완성.
④
사업계획서 작성 (8주, 4/20~)
RISE 아이디어톤 결과물 기반으로 사업계획서 작성. RISE 전형 제출용.
⑤
RISE 사업단 창업동아리 신청 (8주, 4/20~)
RISE 사업계획서 + 팀 구성으로 RISE 사업단 창업동아리 공식 신청.
⑥
전국 아이디어톤 (5월 2주)
연합 교내에서 다듬은 아이디어로 전국 멋사 아이디어톤 참가.
⑦
아이디어 수렴 (5월 4주)
